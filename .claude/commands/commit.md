# Efficient Commit Command for spec-kit

spec-kit のコンテキストを読み取り、適切なコミットメッセージを自動生成してコミットを実行する。

## 実行手順

### 1. 現在のコンテキストを把握する

以下のファイルを順に確認する：

1. `memory/constitution.md` — プロジェクト全体の原則
2. 現在のブランチ名（`git branch --show-current`）から機能番号と機能名を取得
3. 現在の機能に対応する仕様書 `specs/<ブランチ名>/spec.md`
4. タスクリスト `specs/<ブランチ名>/tasks.md` から **完了済み（チェック済み）タスク** を特定

### 2. git status を確認する

```bash
git status
git diff --staged
```

ステージング済みのファイルがない場合は、変更ファイルをユーザーに提示して、どのファイルをステージするか確認する。

ステージング済みのファイルがない場合、以下を実行してステージする：

```bash
git add -A
```

ただし、以下のファイルは **ステージしない**：

- `node_modules/`
- `.env` 系のファイル
- ビルド成果物（`dist/`, `build/`）

### 3. コミットメッセージを自動生成する

以下のフォーマットで生成する：

```
<type>(<scope>): <タスクの内容を日本語で簡潔に記述>

- <変更点1>
- <変更点2>
- <変更点3>

refs: <タスクID（例: T001）>
```

**type の選び方：**

| type       | 使うタイミング                          |
| ---------- | --------------------------------------- |
| `feat`     | 新機能の実装（tasksの新規タスク完了時） |
| `fix`      | バグ修正                                |
| `test`     | テストコードの追加・修正                |
| `refactor` | 機能変更を伴わないリファクタリング      |
| `docs`     | ドキュメント・コメントの変更            |
| `chore`    | 設定ファイル、依存関係の更新            |
| `style`    | コードフォーマットの修正                |

**scope の決め方：**

- ブランチ名の機能番号と機能名から自動決定（例: `001-word-registration` → `word`）
- または変更ファイルのディレクトリ構造から推定

**生成例：**

```
feat(word): 英単語の新規登録APIを実装

- POST /words エンドポイントを追加
- スペル・意味の入力バリデーションを実装
- 登録成功時に201レスポンスを返却

refs: T001
```

### 4. ユーザーに確認する

生成したコミットメッセージを提示し、以下を確認する：

```
以下のコミットメッセージでコミットしますか？

---
<生成したコミットメッセージ>
---

[y] そのままコミット
[e] メッセージを修正してコミット
[n] キャンセル
```

### 5. コミットを実行する

ユーザーの確認後、以下を実行する：

```bash
git commit -m "<生成したコミットメッセージ>"
```

---

## オプション引数

`/commit` コマンドは以下の引数をサポートする：

| 引数         | 説明                           | 例                |
| ------------ | ------------------------------ | ----------------- |
| （引数なし） | 変更内容から自動生成           | `/commit`         |
| `--push`     | コミット後に `git push` も実行 | `/commit --push`  |
| `--amend`    | 直前のコミットを修正           | `/commit --amend` |
| `--wip`      | WIP コミット（後でsquash想定） | `/commit --wip`   |

`--wip` 指定時のコミットメッセージ形式：

```
wip: <変更内容の概要>
```

---

## 注意事項

- `constitution.md` に「コミットメッセージは英語で」などの指定がある場合はそれに従う
- タスクIDが特定できない場合は `refs:` 行を省略する
- spec-kit のブランチ運用（`/specify` 単位でブランチが切られる）を前提とするため、
  `main` / `master` ブランチ上での実行時は警告を出す
