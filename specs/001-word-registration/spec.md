# Feature Specification: 単語登録機能

**Feature Branch**: `001-word-registration`
**Created**: 2026-02-27
**Status**: Draft
**Input**: User description: "単語のwordと意味の登録機能 一つの単語に対して、その時学習した意味を1つ登録できるようにします。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 英単語と意味の新規登録 (Priority: P1)

英語学習中に出会った未知の英単語について、そのスペルと
その時に学習した意味を1つ登録できる。登録することで
学習記録として蓄積し、後から振り返ることができるようにする。

**Why this priority**: 本機能の中核であり、これが無ければ
アプリケーションとして成立しない。単語の登録ができて初めて
照会や復習といった後続機能の価値が生まれる。

**Independent Test**: 単語のスペルと意味を入力して登録ボタンを
押し、登録完了のフィードバックが表示されることで検証できる。

**Acceptance Scenarios**:

1. **Given** 単語登録画面が表示されている状態で,
   **When** ユーザーがスペル欄に "abundant" と入力し、
   意味欄に "豊富な" と入力して登録ボタンを押す,
   **Then** 単語が登録され、登録完了のフィードバックが表示される。

2. **Given** 単語登録画面が表示されている状態で,
   **When** ユーザーがスペル欄に "run" と入力し、
   意味欄に "経営する" と入力して登録ボタンを押す,
   **Then** 既に "run" が "走る" という意味で登録されていても、
   別の学習記録として登録され、登録完了のフィードバックが
   表示される。

3. **Given** 単語の登録が完了した直後,
   **When** 登録完了のフィードバックが表示される,
   **Then** 入力フォームがクリアされ、続けて次の単語を
   登録できる状態になる。

---

### User Story 2 - 入力バリデーション (Priority: P2)

不完全な情報で単語を登録しようとした場合、適切なエラー
メッセージが表示され、ユーザーが入力を修正できる。

**Why this priority**: データの品質を保つために必須だが、
登録機能そのものが先に存在する必要がある。

**Independent Test**: 空欄のまま登録を試み、エラーメッセージが
表示されることで検証できる。

**Acceptance Scenarios**:

1. **Given** 単語登録画面が表示されている状態で,
   **When** スペル欄を空にしたまま登録ボタンを押す,
   **Then** スペルが未入力であることを示すエラーメッセージが
   表示され、単語は登録されない。

2. **Given** 単語登録画面が表示されている状態で,
   **When** 意味欄を空にしたまま登録ボタンを押す,
   **Then** 意味が未入力であることを示すエラーメッセージが
   表示され、単語は登録されない。

3. **Given** 単語登録画面が表示されている状態で,
   **When** スペル欄に空白文字のみを入力して登録ボタンを押す,
   **Then** 有効なスペルが入力されていないことを示すエラー
   メッセージが表示され、単語は登録されない。

---

### Edge Cases

- 同じスペルの単語を異なる意味で複数回登録しようとした場合:
  許容する。同じ単語でも学習した文脈ごとに異なる意味を
  記録するのは学習記録として有効である。
- スペル欄に日本語や記号のみを入力した場合:
  登録を許容する。略語や特殊な表記も学習対象となりうるため、
  英字のみに制限しない。
- 極端に長い文字列（スペルまたは意味）を入力した場合:
  適切な上限を設け、超過時にエラーメッセージを表示する。
- ネットワーク障害などで登録処理が失敗した場合:
  エラーメッセージを表示し、入力内容は保持して再試行
  できるようにする。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ユーザーは英単語のスペルと意味のペアを1件ずつ
  登録できなければならない（MUST）。
- **FR-002**: 登録時にスペルと意味の両方が必須入力で
  なければならない（MUST）。空文字・空白のみの入力は
  受け付けない。
- **FR-003**: 同じスペルの単語を異なる意味で複数回登録
  できなければならない（MUST）。
- **FR-004**: 登録完了後、ユーザーに登録成功のフィードバックを
  表示しなければならない（MUST）。
- **FR-005**: 登録完了後、入力フォームをクリアして次の単語を
  続けて登録できる状態にしなければならない（MUST）。
- **FR-006**: バリデーションエラー時、具体的なエラーメッセージを
  表示しなければならない（MUST）。
- **FR-007**: 各単語には登録日時が自動的に記録されなければ
  ならない（MUST）。
- **FR-008**: スペルおよび意味の入力には適切な文字数上限を
  設けなければならない（MUST）。スペルは200文字以内、
  意味は500文字以内とする。
- **FR-009**: 登録処理が失敗した場合、エラーメッセージを表示し、
  入力内容を保持しなければならない（MUST）。

### Key Entities

- **単語（Word）**: 学習記録の1エントリ。スペル（英単語の
  綴り）、意味（その時に学習した日本語の意味）、登録日時を持つ。
  1つの登録に対してスペルと意味はそれぞれ1つずつ。

### Assumptions

- 本機能は認証済みユーザーのみが利用可能とする（認証機能は
  別機能として実装される前提）。
- 例文の登録は本機能のスコープ外とする（ドメイン情報に
  「例文（任意）」とあるが、別機能として扱う）。
- 学習状況（未学習/学習中/習得済み）の管理は本機能の
  スコープ外とする。
- 登録された単語はユーザーごとに管理される（他のユーザーの
  単語とは分離される）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーがスペルと意味を入力してから登録完了の
  フィードバックを受け取るまでの操作が30秒以内で完了する。
- **SC-002**: 必須項目が未入力の場合、100%の確率でバリデーション
  エラーが表示される。
- **SC-003**: 登録完了後、入力フォームが即座にクリアされ、
  連続して単語を登録する際に追加の操作が不要である。
- **SC-004**: 初めて利用するユーザーが操作説明なしに単語を
  登録できる（直感的な UI）。
